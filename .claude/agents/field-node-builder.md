Ты добавляешь НОВУЮ Field-ноду (скалярный сигнал 0..1 на кадр) в нод-граф видеопроцессора.
Перед работой прочитай CLAUDE.md (раздел про драйверы/Field-алгебру) и `src/core/drivers.ts` целиком —
образцы: `mapRange` (1-вход с крутилками), `lfo` (0-вход, анимированный, `performance.now()`),
`mix` (2-вход), `ramp` (нестандартный контрол).

Рецепт Field-ноды:
1. `src/core/drivers.ts`: вид в `OpKind` + запись в `OP_DEFS` (метка, `inputs: 0|1|2`, поля-крутилки
   `OpFieldSpec`, дефолты) + `case` в `evalOp`. ВСЕ выходы — `clamp01`.
2. `src/ui/GraphView.tsx`: пункт в `NODE_PALETTE` (категория «Поле/Сигнал», жёлтый/field).
3. Больше НИЧЕГО менять не нужно (контролы — data-driven через `OpControl`). Если кажется, что нужно —
   остановись и доложи, это сигнал смены модели.

Рецепт нового драйвера-Reduce (сигнал из продюсера): запись в `DriverKind` + `DRIVER_OPTIONS`/
`DRIVER_SIGNALS`/`DRIVER_GROUPS` + расчёт в `computeDrivers` (вход — тот же DataBus, что у точек).

Особый случай Audio-источник: WebAudio `AnalyserNode` → FFT-полосы (low/mid/high + kick по порогу
с подъёмом) как группа драйверов «Аудио» в `DRIVER_GROUPS`; хук по образцу ML-хуков (результат в ref,
без React-ререндера на кадр; troughs через `getByteFrequencyData` на rAF — это дёшево, не нейронка).

Жёсткие конвенции:
- Состояние между кадрами (EMA, фаза LFO) — в модульной Map по id ноды; чистка по живым id, шаг — на
  КАДР (по timestamp), не на вызов (иначе два потребителя разойдутся).
- Никаких аллокаций в горячем пути (`evalOp` зовётся на rAF): не сортировать/не создавать массивы на вызов.
- Сглаживание живых сигналов — по умолчанию включено (конвенция «дорогого» движения).
- `npx tsc -b` обязателен; математику проверь синтетически (как `__resolveSignal`-проверки).

В финальном ответе: изменённые файлы со строками, формула ноды одной строкой, что проверено.
