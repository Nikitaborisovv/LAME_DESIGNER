# AGENTS.md — субагенты разработки

Проектные субагенты Claude Code (определения — в `.claude/agents/*.md`). Цель — растянуть лимит
подписки (тяжёлая модель только там, где цена ошибки высока — ARCHITECTURE §3.6.10) и закрепить
шаблоны, чтобы рутинные расширения не ломали конвенции. Конвенции кода — [CLAUDE.md](CLAUDE.md),
план — [ARCHITECTURE.md](ARCHITECTURE.md).

| Агент | Модель | Когда использовать |
|---|---|---|
| `fx-pass-builder` | Sonnet | Новый 2D map→map проход по шаблону B1–B4 (трек T-Beauty: Lookup, Displace, Mirror, CA/Grain, Feedback и любые новые Map2D-фильтры) |
| `field-node-builder` | Sonnet | Новая Field-нода по шаблону OP_DEFS/evalOp (Lag, Trigger, Hold…), новый драйвер-Reduce, Audio-источник |
| `config-reviewer` | Opus | Ревью ПОСЛЕ правок конфиг-ядра (`types`/`layerMigrate`/`presets`/`drivers`/`pointSources`) и перед фиксацией крупных фич — чек-лист back-compat пресетов, wiring-поля, instance-id, dispose, rAF-конвенции |

Правила:
- **Не по шаблону → не субагент-строитель.** Всё, что меняет модель данных (GraphDoc/T0e, per-element
  поля, движок частиц T5, миграции) — основная сессия на Opus, со сверкой с ARCHITECTURE §3.6.
- Строители обязаны заканчивать `npx tsc -b`; ревьюер запускается на их результат, если правка
  задела ядро.
- Встроенные `Explore`-агенты — для поиска по коду (дёшево); этот файл не для них.
- Новый агент добавляется только под ПОВТОРЯЮЩИЙСЯ класс задач — не плодить одноразовые.
